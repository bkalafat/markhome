# Security Policy

## Supported Versions

Security fixes target the latest published minor version.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Report security concerns by emailing the maintainer or using GitHub private vulnerability reporting if it is enabled for the repository.

Include:

- affected package
- affected version
- reproduction steps
- expected impact

## Scope

MarkHome parses text and renders SVG. Security-sensitive reports are most likely to involve unsafe browser rendering, SVG injection, CLI file handling, or package publishing integrity.
